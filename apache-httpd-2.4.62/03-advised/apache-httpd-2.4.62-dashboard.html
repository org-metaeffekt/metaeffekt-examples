Security Policy

The security policy used to generate the Vulnerability Assessment Dashboard has been configured as shown below.

CVSS Selector for Provided Vectors

The following CVSS selector is evaluated to find one representative CVSS vector from the unmodified vectors provided from the data sources per CVSS version.
The [Initial Vectors] CVSS Selector contains [1] rule that will be applied in the following order:
  • The first matching vector is selected: [NVD-CNA-NVD], [Microsoft Corporation-*-*], [NVD-CNA-Microsoft Corporation], [GitHub, Inc.-*-*], [NVD-CNA-GitHub, Inc.], [OSV-*-GitHub, Inc.], [OSV-*-*], [NVD-*-*], [CERT-SEI-*-*], [not:Assessment-*-*]
    From the selected vector, [all] vector components are applied to the resulting vector.

CVSS Selector for Effective Vectors

The following CVSS selector is evaluated to find one representative CVSS vector from the effective vectors provided from the data sources per CVSS version.
The [Context Vectors] CVSS Selector contains [4] rules that will be applied in the following order:
  • The first matching vector is selected: [NVD-CNA-NVD], [Microsoft Corporation-*-*], [NVD-CNA-Microsoft Corporation], [GitHub, Inc.-*-*], [NVD-CNA-GitHub, Inc.], [OSV-*-GitHub, Inc.], [OSV-*-*], [NVD-*-*], [CERT-SEI-*-*], [not:Assessment-*-*]
    From the selected vector, [all] vector components are applied to the resulting vector.
  • If present, the [Assessment-*-all] vector is selected.
    From the selected vector, [all] vector components are applied to the resulting vector.
    The following [1] statistics collector will also be applied to the selected vector:
    - If a vector is returned from the selection, [1] is added to the stats collector attribute [assessment].
  • If present, the [Assessment-*-lower] vector is selected.
    From the selected vector, only vector components that lead to a [lower or equal] score on the resulting vector are applied.
    The following [1] statistics collector will also be applied to the selected vector:
    - If a vector is returned from the selection, [1] is added to the stats collector attribute [assessment].
  • If present, the [Assessment-*-higher] vector is selected.
    From the selected vector, only vector components that lead to a [higher or equal] score on the resulting vector are applied.
    The following [1] statistics collector will also be applied to the selected vector:
    - If a vector is returned from the selection, [1] is added to the stats collector attribute [assessment].
After finishing the cvss selection, [1] statistics evaluator will be applied to the resulting vector:
  • If the stats collector attribute [assessment] is [equal] to [0], then [no vector is returned].
Additionally, [1] vector evaluator will be applied to the resulting vector:
  • If the resulting vector is [not base fully defined], then [no vector is returned].

CVSS Version Selection Policy

The versioned vectors selected by the CVSS selectors above are then further reduced to a single vector per selector using the first filter that applies from these.
  • The vector with the highest version number

Vulnerabilities

A vulnerability is mapped to the insignificant status if it is "in review" and has a CVSS overall score below 7.0.

Priority Score

The priority score of a vulnerability is determined by calculating the sum of the following metrics:
  • CVSS Overall: Either the Context CVSS, or as a fallback the Initial CVSS vector is used to calculate the base priority score the other metrics are added to.
  • Keywords: The sum of all matched keyword set scores.
  • EPSS: The exploit probability p of the vulnerability and the parameters min=0.5, f=0.5 and F=2.0 are used in the following formula only if p>=min, otherwise 0 is used: f + ((p - min) / (1.0 - min)) * (F - f)
    This formula represents a truncated linear function between (min, f) and (1, F) with a minimum value of f and a maximum value of F.
  • KEV: If an exploit is known on a vulnerability, exploit=2.0 is added to the priority score. If additionally a use in a ransomware campaign is known, ransomware=1.0 is added.
  • EOL: The end-of-life state of the affected product is evaluated. The priority score is increased by the value of the worst-case state, for all affected components.
    If the product does not provide extended support, the following values are used: supportValid=0.0, supportEndingSoon=3.0, supportExpired=4.0
    If the product provides extended support, the following values are used: supportValid=0.0, supportEndingSoon=1.0, extendedSupportValid=2.0, extendedSupportEndingSoon=3.0, extendedSupportExpired=4.0

The following severity ranges are used to categorize the priority score into severity levels, only if there is an elevation from the base cvss overall score through any of the other metrics.
